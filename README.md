# River_OpenSource

`River_OpenSource` 是一个面向 **内存采集链路工程化实践** 的学习型开源仓库，包含 TCP 版本的 `RiverServer` 与 `RiverClient` 两部分代码。  
本仓库聚焦于网络传输层与采集链路的性能优化研究，目标是为高频读场景提供更低时延、更稳定的吞吐与更清晰的可观测性。

## 项目结构

- `RiverServer/`：服务端（采集端）核心实现与工程文件。
- `RiverClient/`：客户端（调用端）核心实现与工程文件。

## 技术亮点（网络层）

- **低时延链路设计**：围绕小包高频读负载，采用控制面/数据面拆分、连接池与多路复用策略，降低串行阻塞导致的尾延迟。
- **异步并发处理路径**：服务端采用异步收发与固定 worker 池，客户端请求侧支持并发在途窗口，提升单位时间内有效请求处理能力。
- **READSCATTER 优化**：针对离散读请求进行了批量打包与并发调度优化，减少请求管理开销，提升实时刷新场景下的稳定性。
- **缓冲与发送路径优化**：通过响应缓冲池、发送帧复用、发送队列治理等方式减少中间拷贝和频繁分配，提高长期运行稳定性。
- **会话与故障恢复机制**：包含连接超时回收、会话状态重建、重连路径治理与可诊断日志，降低长时间运行时的“假在线/真断连”风险。
- **配置与协议演进能力**：在保持历史代码可回溯的基础上，逐步收敛到 TCP 主路径，便于做 A/B 对照与回归验证。

## 关键优化方向（工程视角）

1. **延迟优先**：优先消除请求链路中的阻塞点（同步等待、队头阻塞、单线程串行瓶颈）。
2. **吞吐优先**：通过并发窗口、批量请求、缓冲复用提升有效带宽利用率。
3. **稳定性优先**：对连接状态机、超时回收、重连流程做显式治理，减少运行时卡死与资源泄漏。
4. **可定位优先**：提供分层日志（链路日志 / 上层调用日志 / 诊断日志），方便快速定位“网络问题 vs 业务问题”。

## 构建说明（Windows / Visual Studio）

- 建议环境：
- `Visual Studio 2022`（含 MSVC 工具链）
- Windows SDK

- 典型构建入口：
- `RiverServer/RiverServerSingleDll.sln`
- `RiverClient/LeechCore.sln`

- 构建配置建议：
- `x64`
- `Release`
- Runtime 视场景选择（例如 `MT` 静态运行库）

> 注：仓库默认保留了便于研究和对照的工程文件；构建产物与中间文件已在根 `.gitignore` 中排除。

## 适用场景（学习研究）

- 低时延网络采集链路设计与性能调优实践
- C/C++ 网络层重构与状态机治理
- 复杂 IO 负载下的并发模型与缓冲策略验证
- 远程采集系统的可观测性与故障恢复工程化
